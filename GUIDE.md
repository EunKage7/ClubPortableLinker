# Club Portable Linker — Гайд по сборке портативок

Практическое руководство: как из обычной установленной программы/лаунчера сделать
портативный пакет на игровом диске и запускать его на бездисковых (CCBOOT) ПК клуба.

---

## 0. Идея за 30 секунд

1. Программа ставится в стандартные места Windows (`C:\Program Files`, `%LOCALAPPDATA%`,
   `C:\ProgramData`, реестр).
2. Линкер **переносит** эти данные в одну портативную папку на игровом диске
   (например `D:\Programs\Steam`) и ставит **junction-ссылки** со старых путей Windows на неё.
3. В корне пакета создаются **`Run.cmd`** (запуск) и **`Stop.cmd`** (корректно закрыть).
   `Run.cmd` на каждом запуске сам наводит ссылки на свой пакет, импортирует `.reg`,
   поднимает службы/задачи и запускает программу.
4. На бездисковом ПК система откатывается при перезагрузке — а `Run.cmd` каждый раз
   возвращает всё на место. Данные при этом лежат на «живом» игровом диске.

> Запуск `Run.cmd` в клубе идёт через **Gizmo client** — он уже даёт права администратора.

---

## 1. Перед началом

- Запускайте линкер **от администратора** (junction, реестр, службы требуют прав).
- Решите, **куда собирать**. Это папка на игровом диске, не на `C:` (на бездиске `C:`
  откатывается). Пример: `D:\Programs\<Имя>` или `D:\OnlineGames\<Имя>`.
- Один пакет = одна папка = один `Run.cmd` + `Stop.cmd`.
- Поля в окне по умолчанию пустые; у каждой кнопки есть подсказка (наведите мышь).

---

## 2. Сценарий A — собрать УЖЕ установленную программу (самый частый)

Программа уже стоит на `C:`. Хотим сделать её портативной.

### Через окно (GUI)

1. Вкладка **«Собрать программу»**.
2. Блок **«Что и куда собираем»**: **Название** (напр. `Steam`) и **Куда собрать**
   (напр. `D:\Programs\Steam`, кнопка `...`).
3. Блок **«Вариант B — из готовой папки»**: укажите **главную папку** программы
   (напр. `C:\Program Files (x86)\Steam`) и нажмите **«Собрать портабл автоматически»**.
4. Линкер сам найдёт связанные папки и ветки реестра, перенесёт всё в пакет, поставит
   ссылки и создаст `Run.cmd` + `Stop.cmd`. Действие необратимое — спросит подтверждение.
5. Проверьте: вкладка **«Применить / Проверить»** → **«Проверить пакет»** (даст вердикт).

### Через командную строку (CLI)

```
ClubPortableLinker.exe --auto-folder --source "C:\Program Files (x86)\Steam" --destination "D:\Programs\Steam" --name Steam
```

Только посмотреть план/размер, без переноса — добавьте `--no-apply`
(или в окне кнопка **«Превью»** / `--preview --source <папка>`).

---

## 3. Сценарий B — собрать ЧЕРЕЗ установщик (чистый захват)

Программа ещё НЕ установлена. Хотим поймать всё, что она создаёт при установке.

1. Вкладка **«Собрать программу»** → блок **«Вариант A — из установщика / URL»**.
2. В поле — путь к установщику **или** прямой URL (линкер скачает сам).
3. **Шаг 1 — снимок и запустить установщик**: снимок системы + запуск установщика.
   **Установите программу как обычно.**
4. **Шаг 2 — собрать портабл**: линкер сравнивает «до/после», находит новые папки и
   ветки реестра, переносит их в пакет и создаёт `Run.cmd`/`Stop.cmd`.

> Вариант A нужен, когда программа кладёт данные в неочевидные места — снимок до/после
> ловит их точно.

## 4. Платформы-лаунчеры (распознаются автоматически)

Линкер знает типовые лаунчеры и собирает их в один клик — нужные папки, данные, запуск
и ветки реестра подставляются сами:

**BlueStacks, Steam, Epic Games, Battle.net, EA App, Riot Client / Valorant, RAGE MP, FACEIT,
Ubisoft Connect, GOG Galaxy, Rockstar Games, Wargaming Game Center, Lesta Game Center
(Мир танков/кораблей), BattleState / Escape from Tarkov, VK Play, 4game (Innova).**

Достаточно указать главную папку (Сценарий A) — остальное линкер добавит сам.

### Каталог лаунчеров (блок «Платформы-лаунчеры» на вкладке «① Создать пакет»)

Внизу вкладки **«① Создать пакет»** — список всех лаунчеров, которые линкер знает, со
значком ✓ у тех, что **установлены на этом ПК** (+ путь). Кнопки:
- **«Сканировать ПК»** — обновить, что найдено (по типовым папкам установки).
- **«Найти и собрать»** — подставить найденную папку выбранного лаунчера в поля сборки на
  этой же вкладке (укажите «Куда собрать» — по умолчанию несистемный диск, напр. `D:` — и
  нажмите сборку).
- **«Сохранить как рецепт»** — сделать из найденного лаунчера переносимый рецепт-шаблон
  (`recipes\<Имя>.json`, пути в `%ProgramFiles%`-токенах) **без переноса данных**. Потом
  его видно на вкладке **«④ Библиотека»** (раздел «Рецепты») и можно применять на любой машине.

### Игры внутри платформы

Клубная схема (проверена на практике): **лаунчер — портабл на D:** (junction C→D
«обманывает систему»), **игры ставятся прямо на общий игровой диск в `D:\Games`**
(этот диск переобраз клиента не трогает). То есть лаунчер и игра живут на D:, а C: —
только ради ссылок.

- **Игры в папке лаунчера** (`Steam\steamapps\common\*`, `Epic Games\<Игра>`) переезжают
  с папкой лаунчера автоматически.
- **Игры в `D:\Games`** (как ставит клуб через лаунчер) физически НЕ переносятся — они уже
  на персистентном диске. Линкеру нужно лишь **затащить регистрацию игры в reg пакета**
  (см. ниже), чтобы лаунчер нашёл её на клиенте.

### Reg конкретной игры — ОБЯЗАТЕЛЬНО для видимости игры на бездисковом клиенте

> **Зачем.** На бездисковом (CCBOOT) клиенте C: каждый раз чистый из образа. Лаунчер
> узнаёт, что игра установлена, из ветки реестра (напр. Ubisoft —
> `HKLM\…\Ubisoft\Launcher\Installs\<id>` с `InstallDir=D:\Games\<игра>`). Эта ветка
> попадает на клиент **только** если она лежит `.reg`-файлом в пакете и `Run.cmd` её
> импортирует. Нет reg-файла → на клиенте будет лаунчер, но **без игры**.

1. Установите/убедитесь, что игра лежит в `D:\Games\<игра>` и лаунчер видит её на ЭТОМ ПК.
2. Вкладка **«③ Игры и reg»** → укажите **Папку игры** = `D:\Games\<игра>` —
   **Название подставится само**. Reg ищется и по имени, и по этому пути (ключи со
   значением `InstallDir`/`InstallLocation = D:\Games\…` тоже попадут).
3. **Снимок reg** (если ставите с нуля) → установка → **Сохранить reg игры** — линкер
   найдёт ветки реестра игры, сохранит их `.reg` в пакет и пересоберёт `Run.cmd`.
   Если игра уже стоит — просто «Сохранить reg игры» по её папке.

Внизу — список **«Игры внутри платформы»** с галочками (включить/выключить игру в сборке),
кнопка **«Сохранить состояние игр»**.

> **На каждую НОВУЮ игру** повторяйте этот шаг — иначе её регистрация не попадёт в пакет
> и на клиенте её не будет в лаунчере.

CLI:
```
:: захватить reg уже установленной игры (путь в D:\Games) в пакет лаунчера
ClubPortableLinker.exe --reg-capture --package "D:\Programs\Ubisoft" --game "Assassin's Creed Syndicate"

:: либо «снимок до / установка / сравнение после» для чистого захвата
ClubPortableLinker.exe --reg-snapshot --snapshot "D:\Temp\before.json"
:: установить игру в D:\Games
ClubPortableLinker.exe --reg-capture --package "D:\Programs\Ubisoft" --game "Имя игры" --snapshot "D:\Temp\before.json"
```

**Проверка «игра увидится на клиенте» (без клиента, на сборочном ПК):** удалите ветку игры
из реестра (`reg delete "HKLM\…\Installs\<id>" /f`), импортируйте reg пакета
(`reg import "D:\Programs\<Лаунчер>\.portable\Registry\<…>.reg"`) — ветка должна вернуться с
путём `D:\Games\…`. Если вернулась → на клиенте `Run.cmd` сделает то же, и игра будет видна.

## 5. Применить готовый пакет / запуск в клубе

- **В клубе:** Gizmo/ярлык зовёт **`Run.cmd`** из корня пакета — он сам наводит ссылки,
  импортирует reg, поднимает службы/задачи и стартует программу. Больше ничего не нужно.
- **Вручную (окно):** «Применить / Проверить» → **«Сделать всё»** (необратимо, спросит
  подтверждение).

Точечные кнопки (блок «Точечно»):
- **План** — показать, что будет сделано, без изменений (безопасно).
- **Ссылки** — применить только junction-ссылки.
- **Реестр** — импортировать только `.reg`.
- **Пересобрать Run.cmd** — перегенерировать `Run.cmd`/`Stop.cmd` (НЕ запускает программу).
- **Открыть reg** — папка с reg-файлами.

CLI:
```
ClubPortableLinker.exe --apply-package --package "D:\Programs\Epic"
ClubPortableLinker.exe --apply-package --package "D:\Programs\Epic" --safe   :: только reg+batch, без ссылок
```

> **Самовосстановление ссылок.** `Run.cmd` при каждом запуске сам наводит junction на
> СВОЙ пакет: если ссылка ведёт не туда — пересоздаёт; если по пути реальная папка —
> отодвигает её в `.bak` и ставит ссылку. Поэтому **несколько версий портабла** можно
> держать рядом и переключаться запуском нужного `Run.cmd` — ссылки не «дерутся».
> Отдельная «починка ссылок» не нужна.

---

## 6. Закрыть платформу перед переносом/удалением

Чтобы папку пакета можно было перенести/удалить (не «занята»):

- Двойной клик по **`Stop.cmd`** в корне пакета — он остановит службы пакета и закроет
  все его процессы (лаунчер + дочерние). После этого папка свободна.

---

## 7. Проверка перед выдачей в клуб

- **Окно:** «Применить / Проверить» → **«Проверить пакет»** — покажет ссылки (OK/нет/не
  ссылка), reg-файлы, состояние `Run.cmd` и **итог-вердикт** «исправен / проблем N».
- **CLI:**
```
ClubPortableLinker.exe --verify-package --package "D:\Programs\Epic" [--json]
```
Коды возврата: `0` — ок, `2` — есть проблемы, `1` — ошибка запуска.

---

## 8. Упаковать пакет в архив (перенос на другой ПК)

- **Окно:** «Применить / Проверить» → **«Запаковать в ZIP»** (пакует в фоне, с полосой
  прогресса, по готовности открывает папку).
- **CLI:**
```
ClubPortableLinker.exe --export-zip --package "D:\Programs\Epic" --out "D:\Archives\Epic-portable.zip"
```

В архив **не** попадают служебные `_Replaced`, `_Backups`, `.portable\BatchBackups` и
junction-папки. При распаковке появляется папка с именем пакета — запускайте `Run.cmd`
из неё.

## 9. Рецепты (переносимые шаблоны)

**Рецепт** — это сохранённый шаблон сборки приложения (с токен-путями `%ProgramFiles%`,
`%LOCALAPPDATA%` — переносим между машинами и пользователями). Готовые рецепты
**Epic / Battle.net / EA** уже лежат в папке `recipes\` рядом с программой.

Вкладка **«④ Библиотека»**, раздел **«Рецепты»** (всё локально, рядом с exe):
- **Создать из пакета** — сохранить открытый пакет как рецепт в `recipes\`.
- **Применить рецепт** — выбрать рецепт → указать папку → линкер создаст там пакет;
  затем «Сделать всё».
- **Папка рецептов** — открыть `recipes\`.

Список обновляется сам при входе на вкладку.

**Перенос между серверами.** Рецепты лежат в `recipes\` рядом с exe — копируешь папку
линкера на другой сервер, рецепты едут вместе. Хочешь общую папку на сервере — держи
рецепты там и тяни из CLI: `--update-recipes --shared "\\SERVER\recipes"` (с бэкапом).

**Обновление с откатом.** При перезаписи рецепта прежняя версия сохраняется в
`recipes\_history\<имя>.<дата>.json` — если новая версия что-то сломала, вернитесь к старой.

CLI:
```
ClubPortableLinker.exe --save-recipe --package "D:\Programs\Steam"
ClubPortableLinker.exe --list-recipes
ClubPortableLinker.exe --apply-recipe --recipe "recipes\Epic.json" --destination "D:\Programs\Epic" [--apply]
ClubPortableLinker.exe --update-recipes --shared "\\SERVER\recipes"   :: подтянуть из общей папки
```

> Если рядом с exe писать нельзя (линкер в `Program Files`) — рецепты уходят в
> `%LOCALAPPDATA%\ClubPortableLinker\recipes`.

## 10. Каталог пакетов

Вкладка **«④ Библиотека»**, раздел **«Каталог пакетов»** показывает готовые пакеты (папки с
`.portable\manifest.json`) из **папок поиска**. У каждого сервера своя раскладка, поэтому:
- кнопка **«Добавить папку»** — укажи корень, где лежат твои сборки (запоминается в
  `linker.settings.json`); линкер сканит внутри неё;
- папки **собранных/открытых** пакетов запоминаются **сами** — каталог «учится»;
- **«Сканировать»** — обновить сейчас; обновляется и при входе на вкладку.

Двойной клик по строке — открыть пакет. Скан идёт в фоне, окно не виснет.

---

## 11. Дозахват докачанной игры (дельта)

Если на готовую платформу докачали игру и её данные легли в новую папку:
```
ClubPortableLinker.exe --fs-snapshot --out "D:\Temp\before.json"   :: ДО докачки
:: докачать игру
ClubPortableLinker.exe --capture-delta --package "D:\Programs\Steam" --snapshot "D:\Temp\before.json"          :: показать новое
ClubPortableLinker.exe --capture-delta --package "D:\Programs\Steam" --snapshot "D:\Temp\before.json" --apply   :: занести в пакет
```

## 12. Что внутри готового пакета

```
D:\Programs\Epic\
  Run.cmd                       ← запуск + самовосстановление ссылок (его зовёт Gizmo)
  Stop.cmd                      ← корректно закрыть (службы + процессы) перед переносом
  ProgramFiles\ ...             ← перенесённые данные программы
  ProgramFilesX86\ ...
  ProgramData\ ...
  Local\  Roaming\ ...          ← %LOCALAPPDATA% и %APPDATA%
  .portable\                    ← служебное (скрытое)
    manifest.json               ← описание пакета (ссылки, reg, игры)
    Registry\                   ← .reg для импорта при запуске
      <платформа>.reg
      Games\<игра>\*.reg        ← reg конкретной игры
  _Replaced\                    ← сюда уехали дубли путей Windows (можно удалять)
```

> `Run.cmd` импортирует **все** `.reg` из папок `.portable\Registry`, `Registry`, `Reg`.
> Если обновление игры поменяло путь — правьте `.reg` прямо в этих папках, пересборка не нужна.
>
> ⚠️ **Доверие:** `.reg` импортируются с правами администратора при каждом запуске.
> Кладите в эти папки только **свои/проверенные** reg-файлы и не запускайте чужие пакеты
> из непроверенных источников — подсунутый `.reg` применится к реестру системы.

## 13. Частые вопросы

- **Лаунчер «не видит» игру / пусто.** Не хватает одного из трёх: reg игры, data-папок
  платформы, самой папки игры. Сделайте reg-захват игры (раздел 4) и проверьте, что папки
  платформы вошли в пакет.
- **«Папка занята», не переносится/не удаляется.** Сначала `Stop.cmd` (раздел 6).
- **Окно «задумалось».** Тяжёлые операции (скачивание, снимок, проверка, сборка, упаковка)
  идут в фоне с полосой прогресса — дождитесь; повторный запуск блокируется до конца.
- **Не собирать с `C:`.** Источник может быть на `C:`, но «Куда собрать» — всегда игровой диск.
- **Огромные платформы (Steam).** Сначала «Превью»/`--no-apply`, оцените размер, потом применяйте.

## 14. Шпаргалка CLI

| Команда | Что делает |
| --- | --- |
| `--auto-folder --source <папка> --destination <куда> --name <имя>` | Собрать из установленной программы |
| `--auto-folder ... --no-apply` | Только план, не переносить |
| `--preview --source <папка>` | Что и какого размера попадёт в портабл (без переноса) |
| `--reg-snapshot --snapshot <файл>` / `--reg-capture --package <пакет> --game <игра> --snapshot <файл>` | Reg игры (до/после) |
| `--fs-snapshot --out <файл>` / `--capture-delta --package <пакет> --snapshot <файл> [--apply]` | Дозахват новых папок (дельта) |
| `--apply-package --package <пакет>` / `... --safe` | Применить пакет (без ссылок — `--safe`) |
| `--verify-package --package <пакет> [--json]` | Проверить пакет |
| `--export-zip --package <пакет> --out <zip>` | Упаковать в архив |
| `--save-recipe --package <пакет>` / `--list-recipes` / `--apply-recipe --recipe <json> --destination <куда> [--apply]` / `--update-recipes` | Рецепты |
| `--plan --config <пакет> --profile <имя>` | План по профилю |

Полный список — `ClubPortableLinker.exe --help`.
